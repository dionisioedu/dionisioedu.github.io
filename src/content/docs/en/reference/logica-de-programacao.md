Programming logic is the ability to convert a problem into a finite, executable sequence of steps.

Language matters, but logic matters more at the beginning.

## What you must master

### Sequence

Execute instructions in the correct order.

### Decision

Make the program choose paths based on conditions (`if`, `else`, `switch`).

### Repetition

Repeat actions with control (`for`, `while`) without creating infinite loops.

### Decomposition

Break large problems into smaller parts (functions).

## A practical mental model

Always answer these 4 questions:

1. What is the input?
2. What transformation should happen?
3. What is the expected output?
4. How do I verify correctness?

If you answer these before coding, your error rate drops fast.

## Recommended flow before coding

1. Write the rule in plain language.
2. Break it into numbered steps.
3. Create input/output examples.
4. Only then implement.

This process saves time and reduces rework.

## Common mistakes

- coding before understanding the problem;
- mixing many responsibilities in one function;
- ignoring edge cases;
- relying only on trial and error.

## High-value exercises

1. Check if a number is even or odd.
2. Compute the average from a list of grades.
3. Validate a password against minimum rules.
4. Find min and max values in a collection.
5. Build a simple interactive menu.

For each exercise, do 3 stages:

- pseudocode;
- implementation;
- readability review.

## How to know you are improving

- You can explain your code out loud.
- Your functions get smaller and clearer.
- You solve similar problems faster over time.
- You stop memorizing and start understanding.

That is the turning point for every strong beginner.
